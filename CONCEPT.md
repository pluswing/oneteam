# one team

one teamは単独開発者向けのAI開発アシストツールです。  
GitHubライクな、issue, pull request機能を持ち、  
gitで管理されたソースコードを管理できるUIツールを持ち、  
issueに書かれた内容は、AIが自動で開発のための要件定義を行い、  
実装、テストを自動で行い、pull requestを出すところまで半自動で行います。  
pull requestに出された内容については、これもAIが自動でレビューを行い、  
不備があれば、自動で修正、テストを行います。  

## UI
Githubのissue, pull request機能を持ちます。
デザインはGithubに寄せてください。

- issue機能
  - issueの一覧
  - issueの新規作成/更新/削除
  - 状態管理 ( Open / Closed )
  - タグ管理 ( 要件定義中 / 実装待ち /  実装中 / テスト中 / 完了 など。 )
    - 基本的にAIが次に行う仕事を決めるのに利用する。
  - コメント機能
    - AIからの要件定義のための質問
    - 人間からの回答
    - 実装を始めます。や実装完了。などのステータス変更時メッセージ
      - などに利用する
- pull request機能
  - pull requestの一覧
  - pull requestの新規作成/更新/削除
  - 状態管理 ( Open / Closed )
  - タグ管理 ( レビュー中 / 修正中 / テスト中 / 完了 など。 )
  - コメント機能
    - タグとコメントは、issueとほぼ同じ
  - コミット一覧
  - 変更ファイルの確認

## AI
以下のAIを実装し、開発、テストなどを自動で行う。

- 要件定義エージェント
  - issueに特定のタグがついたら、このエージェントが動く。
  - issueに書かれた内容で不明確な部分や既存コードと折り合いのつかない部分などを指摘し、確認を行う
  - 人間のコメントを待ち、全ての指摘がクリアになるまでやりとりを続ける
  - 要件定義が完了したら、コメントに定義された要件を書き込み、実装エージェントにバトンタッチする
- 実装エージェント
  - 要件定義に沿って実装を行う。
  - 既存コードを確認しながら、開発・修正を行う。
  - 実装時にブランチを切り、開発を行い、実装中に不明確な点が出てきた場合、issueのコメントで確認を行う。
  - 人間のコメントを待ち、実装を行うということを開発が完了するまで繰り返す。
  - ユニットテストは適宜開発を行い、動作確認を行いながら開発を行う。
  - 開発が完了したら、pull requestを出す。
- レビューエージェント
  - 作成されたpull requestに対しレビューを行う。
  - 既存コードのルールを守っているかなどを確認する。
  - 指摘項目がある場合、指摘項目をコメントに書き込み、修正エージェントにバトンタッチする
  - 問題がない場合は、QAエージェントにバトンタッチする
- 修正エージェント
  - 指摘事項の修正を行う。
  - 修正が完了し、テストが全てパスすることを確認したら、再度レビューエージェントにバトンタッチする。
- QAエージェント
  - UIを含めたQAを実施するエージェント
  - 実行環境を自動で用意し、環境を立ち上げる
  - playwright などを利用し、追加・修正された機能をテストする
  - 不具合などがあった場合、コメントに不具合の詳細を書き込み、修正エージェントにバトンタッチ
  - 問題なければ、検証結果をまとめてコメントし、完了タグをつける

# バックエンド、動作環境
one teamは単独開発者向けのシステムです。
ポータブルに動作させたいので、立ち上げはシンプルに行えるようにしてください。
dockerは使用せず、nodejsとgitさえインストールされていれば動作するようにしてください。
データベースはlibSQLを利用してください。
UIのポートはユーザが決められるよう設定ファイルを用意してください。
git repositoryはone team内の特定の場所に作成される、もしくは初回起動時にインポートを行うイメージです。
